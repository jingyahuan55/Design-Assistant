## Demo 建议输入

### Case 1

主题：
`新加坡食阁`

目标：
验证主题到视觉语言的转译是否成立。

### Case 2

主题：
`包容性社区医疗服务`

目标：
验证可访问性和柔和专业风格是否能体现。

### Case 3

主题：
`城市夜间交通信息屏`

目标：
验证高对比度和叠字可读性建议是否合理。

## Demo 讲述顺序

1. 首页介绍产品价值
2. 在 Workspace 输入主题与图片
3. 点击生成设计语言
4. 展示结果页
5. 强调三件事：
   - 它能把主题翻译成视觉方向
   - 它能处理图片与 UI 的适配问题
   - 它能输出可直接复制的 Tokens

## 演示时要重点说的差异点

1. 不是直接生成 UI，而是辅助设计决策
2. 同时支持文字和图片输入
3. 输出结构化建议而不是只有灵感描述
4. 有基础可访问性提醒
5. 有 Token 化输出，便于落地

## 当天收尾建议

1. 保留至少 2 组稳定示例
2. 不要在最后 1 小时重构
3. 把演示重点放在“核心价值已跑通”
4. 如果个别高级功能没做完，明确说明是 V1.5
