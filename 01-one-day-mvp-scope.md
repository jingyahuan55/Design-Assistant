# 01 One-Day MVP Scope

## 产品目标

在 1 天内交付一个可演示的网页 MVP，证明这个产品的核心价值：

`把主题和图片，快速翻译成可落地的界面设计语言。`

## 当天只做什么

### 必做功能

1. 新建分析任务
2. 文字输入
3. 单张图片上传
4. 调用多模态模型进行一次分析
5. 返回固定结构的结果
6. 结果页展示以下模块：
   - 主题摘要
   - 视觉关键词
   - 配色系统
   - 图片适配建议
   - 基础可访问性提醒
   - Token 导出

### 可以弱化的功能

1. 历史记录
2. 版本管理
3. 引导式微调
4. Split View 对比
5. 多图片分析
6. 多用户系统

说明：
这些功能都重要，但不适合在 1 天 MVP 内优先实现。当天先保留接口预留和文档约束，不一定实现完整 UI。

## 当天不做什么

1. 自动生成完整 UI 页面
2. Figma 插件
3. 多人协作
4. 完整项目管理
5. 大型素材库
6. 完整设计系统管理
7. 复杂版本 diff 系统

## 建议的最小页面范围

1. `Landing`
2. `Workspace`
3. `Result`

说明：
当天可以先不单独开发 `History` 和 `Task Detail` 页面，只在代码结构上预留。

## 当天建议保留的数据结构

最少保留 4 个对象：
1. `task`
2. `input`
3. `asset`
4. `analysisResult`

## 当天建议完成的结果结构

```text
result
├─ summary
├─ moodboard
├─ colorSystem
├─ imageAdaptation
├─ accessibility
└─ exportSnippets
```

## 成功标准

### 产品层

用户看完结果后，能理解：
1. 这个主题适合什么视觉方向
2. 该用什么主色和辅助色
3. 这张图适合怎么叠字和上遮罩
4. 结果能不能直接拿去做设计或代码试验

### 技术层

1. 输入到输出跑通
2. 结构化结果稳定
3. 页面能正常展示
4. 图片上传可用
5. 能导出基础 Token

### 展示层

1. 首页能解释产品价值
2. 结果页有清晰层次
3. 结果不只是纯文本堆叠
